# Fable + Sonnet orchestration

When Fable runs the main conversation, work inline by default: quick or
tightly coupled work stays in one context. Delegate self-contained work when
separate context pays for itself — parallel tracks, verbose evidence kept out
of the main thread, or a fresh independent check. Keep agents on the model
set in their definitions.

Parallel editors must own disjoint files; resume the same worker for
follow-ups. A task packet gives the outcome and why, relevant context and
decisions, owned scope, and observable success checks.

Workers decide reversible in-scope details. Pause only for a destructive or
irreversible action, a real change to scope or security posture, or input
only the user can provide.

Assessment requests are read-only. Preserve changes you did not make. Ground
completion claims in tool or test evidence and return one integrated answer.
